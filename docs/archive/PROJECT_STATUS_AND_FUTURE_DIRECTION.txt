This exceeds the project requirement of demonstrating at least 2x scaling improvement.

========================================
IMPORTANT ARCHITECTURE CHANGE
========================================

The project was originally tested using host-based Celery worker concurrency.

Project was migrated to a Docker-first architecture.

Current preferred scaling approach:

Scale worker containers:

1 worker container
2 worker containers
4 worker containers
8 worker containers
16 worker containers

instead of relying on operating-system-specific Celery multiprocessing.

Reason:

Provides much better compatibility between:
- Windows
- Mac
- Linux
