Worker comparison

Final benchmark results for an identical 40-task route-risk workload
Workers Runtime Throughput Speedup Result
1 32.11 seconds 1.25 tasks/sec 1.00× Success

Primary finding

Increasing the worker count from 1 to 8 reduced runtime from 32.11 seconds to 4.37 seconds. This represents approximately 7.35× speedup and about 91.8% parallel efficiency at eight workers.

Benchmark method

Workload

Forty route-risk analysis tasks representing the final Nevada workload were used for each benchmark run.

Architecture

FastAPI submitted work through Redis to Celery workers running in the project Docker environment.

Controlled comparison

The task count and workload remained constant while only the number of active workers changed.
